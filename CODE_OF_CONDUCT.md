# Code of Conduct

QA-Z is a developer tool for evidence-first agent workflows. Project spaces should stay useful, specific, and respectful.

## Standards

- Assume good intent, but ask for evidence when claims affect users.
- Keep critique focused on code, docs, tests, artifacts, and reproducible behavior.
- Do not harass, threaten, dox, or target people.
- Do not post live secrets, private exploit details, or unrelated promotion.

## Enforcement

Maintainers may edit, hide, lock, or remove comments and issues that violate these expectations. Serious or repeated abuse may lead to a ban from project spaces.

Security issues should follow [SECURITY.md](SECURITY.md), not public argument threads.
