# Case Study Seeds

QA-Z needs concrete adoption stories to move from tool to category leader.

## Seed 1: Auth Refactor

An agent simplified authorization. QA-Z caught a non-owner access regression, produced a repair prompt, and verified the candidate improved.

## Seed 2: Security Scan

An agent introduced a risky flow. Semgrep found it, QA-Z normalized the finding, and SARIF plus repair prompts gave reviewers a deterministic next step.

## Seed 3: Pull Request Gate

A team installed the QA-Z GitHub Action. PRs with generated code now preserve fast/deep artifacts before deciding whether the change can merge.

Each case study should include commands, artifacts, before/after evidence, and a clear statement of what QA-Z did not automate.
