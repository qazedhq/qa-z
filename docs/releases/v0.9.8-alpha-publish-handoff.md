tests, CLI help smoke checks, QA-Z fast/deep/benchmark, package build, artifact
install smoke, and bundle manifest verification with one JSON result and one
exit code. Treat the gate JSON as the authoritative final local artifact

## Publish Sequence

Before the GitHub repository exists, run the local-only preflight:

```bash
python scripts/alpha_release_preflight.py --skip-remote
python scripts/alpha_release_preflight.py --skip-remote --json
python scripts/alpha_release_preflight.py --skip-remote --json --output dist/preflight-local.json
```

The preflight JSON includes repository, origin, branch, tag, and mode inputs so
operator evidence can be replayed without guessing which flags were used.
The preflight JSON records check_count, passed_count, failed_count, skipped_count, and failed_checks so a remote blocker can be triaged without scraping console text.
For remote blockers it also emits `next_actions`, including:

- Create or expose the public GitHub repository qazedhq/qa-z, then rerun remote preflight for the intended repository URL.
- Set --repository-url to https://github.com/qazedhq/qa-z.git, or update --expected-repository if a different public GitHub repository is intentional.
- Set origin to the intended repository URL, then rerun preflight with --expected-origin-url.
- Remote already has refs; choose the release PR path with --allow-existing-refs or publish to an empty repository.
- Remote release tag v0.9.8-alpha already exists; inspect the existing tag before publishing a new alpha tag.

After `qazedhq/qa-z` exists as a public GitHub repository and is reachable:

```bash
python scripts/alpha_release_preflight.py --repository-url <repository-url>
python scripts/alpha_release_preflight.py --repository-url <repository-url> --json
git ls-remote --refs <repository-url>
git remote add origin <repository-url>
git remote -v
git status --short
```

Do not add `origin` until this command succeeds for the intended repository URL.
The preflight fails unless GitHub API metadata reports a public `qazedhq/qa-z`
repository and `git ls-remote --refs` returns no refs.
By default it also pins the expected GitHub full name to `qazedhq/qa-z`; use
`--expected-repository <owner/repo>` only if the public target is intentionally
changed before launch and all release docs are updated to match.
After `origin` is configured, rerun preflight with the explicit origin contract:

```bash
python scripts/alpha_release_preflight.py --repository-url <repository-url> --expected-origin-url <repository-url>
python scripts/alpha_release_preflight.py --repository-url <repository-url> --expected-origin-url <repository-url> --json
python scripts/alpha_release_gate.py --include-remote --repository-url <repository-url> --json
```

`python scripts/alpha_release_gate.py --include-remote` defaults the expected
origin URL to the repository URL. HTTPS and SSH GitHub URL forms for the same repository are accepted.

If `qazedhq/qa-z` already has a default branch or bootstrap commit and you are
intentionally using the release PR path, use the explicit existing-ref preflight
instead:

```bash
python scripts/alpha_release_preflight.py --repository-url <repository-url> --allow-existing-refs
python scripts/alpha_release_preflight.py --repository-url <repository-url> --allow-existing-refs --json
git ls-remote --refs <repository-url>
git remote add origin <repository-url>
git remote -v
git status --short
```

The `--allow-existing-refs` option only relaxes the empty-remote check for this
PR path. It still requires the local release branch, clean tracked worktree,
absent local release tag, generated artifact hygiene, a reachable remote, and
GitHub API metadata for a public `qazedhq/qa-z` repository.
It does not allow an existing remote `v0.9.8-alpha` tag.
Use `--json` on the same preflight command when recording machine-readable
operator evidence; it returns the same exit code as the human-readable output.

If `git ls-remote --refs <repository-url>` returns no refs for a new empty
repository, push the current release baseline to the default branch first:

```bash
git push -u origin HEAD:main
```

If the repository already has a default branch or bootstrap commit, push the
release branch and open the release PR instead:

```bash
git push -u origin codex/qa-z-bootstrap
```
