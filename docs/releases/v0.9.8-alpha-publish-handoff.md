## Publish Sequence

Before the GitHub repository exists, run the local-only preflight:

```bash
python scripts/alpha_release_preflight.py --skip-remote
```

After `qazedhq/qa-z` exists and is reachable:

```bash
python scripts/alpha_release_preflight.py --repository-url <repository-url>
git ls-remote --refs <repository-url>
git remote add origin <repository-url>
git remote -v
git status --short
```

Do not add `origin` until this command succeeds for the intended repository URL.

If `git ls-remote --refs <repository-url>` returns no refs for a new empty
repository, push the current release baseline to the default branch first:

```bash
git push -u origin HEAD:main
```

If the repository already has a default branch or bootstrap commit, push the
release branch and open the release PR instead:

```bash
git push -u origin codex/qa-z-bootstrap
```

Open the release PR with:

- title: `Release QA-Z v0.9.8-alpha`
- body: `docs/releases/v0.9.8-alpha-pr.md`

Wait for remote CI before merging. Required evidence:

- `test` job passes
- `Build package artifacts` step passes inside the `test` job
- `qa-z` job passes
- SARIF upload is attempted; permission-related code-scanning upload failures may
  remain `continue-on-error`

After remote CI passes, tag from the validated default branch or merged release
baseline. Prefer a signed tag:

```bash
git tag -s v0.9.8-alpha -m "QA-Z v0.9.8-alpha"
git tag -v v0.9.8-alpha
git push origin v0.9.8-alpha
```

If tag signing is not configured for this alpha, use an annotated tag and record
the reason in operator notes:

```bash
git tag -a v0.9.8-alpha -m "QA-Z v0.9.8-alpha"
git push origin v0.9.8-alpha
```

Create the GitHub release using:

- tag: `v0.9.8-alpha`
- title: `QA-Z v0.9.8-alpha`
- body: `docs/releases/v0.9.8-alpha-github-release.md`

## Safety Boundaries

- Do not commit root `.qa-z/**`.
- Do not commit `benchmarks/results/**` unless intentionally freezing evidence
  with surrounding context.
- Do not start post-release roadmap work before the release baseline has remote
  CI evidence.
- Confirm README, license detection, `SECURITY.md`, issue templates, PR template,
  Actions, code scanning or SARIF artifact behavior, and repository community
  checklist immediately after the public release.
- Do not claim live Codex, Claude, remote orchestration, branch mutation, or
  GitHub bot-comment automation in this alpha.
