## Publish Sequence

After the intended GitHub repository exists and is reachable:

```bash
git ls-remote --heads <repository-url>
git remote add origin <repository-url>
git remote -v
git status --short
git push -u origin codex/qa-z-bootstrap
```

Do not add `origin` until this command succeeds for the intended repository URL.

Open the release PR with:

- title: `Release QA-Z v0.9.8-alpha`
- body: `docs/releases/v0.9.8-alpha-pr.md`

Wait for remote CI before merging. Required evidence:

- `test` job passes
- `Build package artifacts` step passes inside the `test` job
- `qa-z` job passes
- SARIF upload is attempted; permission-related code-scanning upload failures may
  remain `continue-on-error`

After merge, tag from the merged release baseline:

```bash
git tag -a v0.9.8-alpha -m "QA-Z v0.9.8-alpha"
git push origin v0.9.8-alpha
```

Create the GitHub release using:

- tag: `v0.9.8-alpha`
- body: `docs/releases/v0.9.8-alpha-github-release.md`

## Safety Boundaries

- Do not commit root `.qa-z/**`.
- Do not commit `benchmarks/results/**` unless intentionally freezing evidence
  with surrounding context.
- Do not start post-release roadmap work before the release branch has remote CI
  evidence.
- Do not claim live Codex, Claude, remote orchestration, branch mutation, or
  GitHub bot-comment automation in this alpha.
