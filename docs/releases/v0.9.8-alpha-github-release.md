# QA-Z v0.9.8-alpha

QA-Z `v0.9.8-alpha` is the first public alpha release candidate for the
Codex-first, model-agnostic QA control plane in this repository. The matching
Python package version is `0.9.8a0`.

## Highlights

- Deterministic `fast` and `deep` QA gates with repairable local artifacts.
- Review packet and `repair-prompt` generation for failed fast checks and deep findings.
- Repair-session manifests, executor guides, post-repair verification, and publish summaries.
- GitHub Actions summary rendering plus SARIF output for deep findings.
- Benchmark corpus covering Python, TypeScript, mixed fast/deep handoff aggregation, executor bridge action context, executor-result ingest, and live-free executor dry-run policies.
- Self-inspection, backlog, selected-task, and autonomy planning artifacts for local operator handoff.
- Executor bridge packaging and executor-result ingest without live agent execution.
- Frozen pre-live executor safety package and local live-free safety dry-run diagnostics.

## Generated Artifact Policy

Root `.qa-z/**`, `benchmarks/results/work/**`,
`benchmarks/results/summary.json`, and `benchmarks/results/report.md` stay local
by default. Commit those files only when a release intentionally freezes evidence
with surrounding context.

Fixture-local `.qa-z/**` evidence under `benchmarks/fixtures/**/repo/.qa-z/**`
can remain tracked when it is part of a benchmark fixture contract.

## Known Non-Goals

- No live Codex or Claude execution.
- No autonomous code editing.
- No remote orchestration, queues, schedulers, or daemons.
- No branch mutation, commits, pushes, or GitHub bot comments from QA-Z itself.
- No LLM-only judgment replacing deterministic pass/fail gates.
