# QA-Z v0.9.8-alpha

QA-Z `v0.9.8-alpha` is the first public alpha release of the Codex-first,
model-agnostic QA control plane for coding-agent workflows.

The matching Python package version is `0.9.8a0`.

## Release Commit

`6ed3a205443498f4bd063a97106fd9d3a05b99a0`

## Highlights

- Deterministic `init`, `plan`, `fast`, `deep`, `review`, and `repair-prompt`
  command surfaces.
- Fast and deep QA gates that produce repairable local artifacts.
- Review packet and repair-prompt generation for failed fast checks and deep
  findings.
- Repair-session manifests, executor bridge packaging, executor-result ingest,
  and post-repair verification.
- GitHub Actions summary rendering and SARIF output for deep findings.
- Benchmark fixtures covering Python, TypeScript, mixed fast/deep handoff
  aggregation, executor bridge action context, executor-result ingest, and
  live-free executor safety dry-run policies.
- `qa-z doctor` and config validation for launch-readiness checks.
- `qa-z init` profiles plus optional agent template and GitHub workflow
  generation.
- Packaged templates and installed-artifact smoke coverage.
- Built-in timeout defaults for configured checks.
- Targeted fixes for alpha blockers:
  - hardened contract resolution so README/generic markdown files are not
    treated as QA contracts;
  - fixed planner support for the current `deep.checks` config shape;
  - preserved configured commands in smart selection;
  - fixed targeted Semgrep command construction so default scan roots are not
    accidentally retained;
  - fixed temp path style handling for Windows-style temp roots on POSIX CI
    runners.

## Generated Artifact Policy

Root `.qa-z/**`, `benchmarks/results/work/**`,
`benchmarks/results/summary.json`, and `benchmarks/results/report.md` stay local
by default.

Fixture-local `.qa-z/**` evidence under `benchmarks/fixtures/**/repo/.qa-z/**`
can remain tracked when it is part of a benchmark fixture contract.

## Known Non-Goals For This Alpha

- No live Codex, Claude, or other agent execution.
- No autonomous code editing.
- No remote orchestration, queues, schedulers, or daemons.
- No branch mutation, commits, pushes, or GitHub bot comments from QA-Z itself.
- No LLM-only judgment replacing deterministic pass/fail gates.
- No PyPI/TestPyPI/package registry publish in this GitHub prerelease step.
