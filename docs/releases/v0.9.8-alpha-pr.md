# Release QA-Z v0.9.8-alpha

## Scope Summary

QA-Z `v0.9.8-alpha` packages the current Codex-first, model-agnostic QA control
plane baseline for local coding-agent workflows.

This release includes deterministic fast and deep QA gates, review and
repair-prompt artifacts, repair sessions, post-repair verification, GitHub
summary rendering, a benchmark corpus, self-inspection and autonomy planning
artifacts, executor bridge packaging, executor-result ingest, and a live-free
executor safety dry-run.

The matching Python package version is `0.9.8a0`.

## Known Non-Goals

- No live Codex or Claude execution.
- No autonomous code editing.
- No remote orchestration, queues, schedulers, or daemons.
- No branch mutation, commits, pushes, or GitHub bot comments from QA-Z itself.
- No LLM-only judgment replacing deterministic pass/fail gates.

## Generated Artifact Policy

Root `.qa-z/**`, `benchmarks/results/work/**`,
`benchmarks/results/summary.json`, and `benchmarks/results/report.md` stay local
by default. Commit those files only when a release intentionally freezes evidence
with surrounding context.

Fixture-local `.qa-z/**` evidence under `benchmarks/fixtures/**/repo/.qa-z/**`
can remain tracked when it is part of a benchmark fixture contract.

## Benchmark Snapshot

The current benchmark snapshot is `50/50 fixtures, overall_rate 1.0`.

The benchmark corpus covers Python and TypeScript fast failures, mixed
fast/deep handoff aggregation, executor bridge action-context packaging,
